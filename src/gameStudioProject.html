A1 Game Studio Project

An Unreal 4 Group Project

The Project

The Game Studio Project was a group project with our sister degree Game Design. Our team consisted of one programmer (me) and five game designers and we had to make a vertical slice demo for a small game.

We chose Unreal as it allowed us to convert one of the game designers into a programmer by letting him use blueprints. I would implement core features using C++, while the game designers would extend those features using blueprints.

The Game

Our game was a clone of Risk of Rain 2, a rouge lite third-person shooter where the more items you collect the more powerful you become. Each item in Risk of Rain 2 has different abilities that can be triggered by various events.

Our game also has this system where events are fired from the C++ code to blueprints.

The Cel Shader

The Cel Shader is by far the thing I’m most proud about. This really made our game look less like an Unreal tutorial project and more like our own game.

The Cel Shader is a post-processing stack effect at the end of the pipeline. It uses a sobel operator on the normal and depth buffer to detect edges. The depth buffer is good at drawing lines between two objects while the normal buffer is useful for drawing lines where there’s a sudden change within an object. Colour banding is also used to give it a more cartoon look.

Blueprints and C++

There was a steep learning curve to learning Unreal compared to Unity. However, it was worth it overall as the designers could help out using blueprints.

The core mechanics of the game are dealt with in C++, such as enemies, loot and components, but more creative things, like particles, are controlled with blueprints.

One major crossover between the two was items. Items in our game are components that can be attached to any actor and do some sort of effect based on a set of triggers. Yes, they were fairly abstract. A simple example is an item that has a chance to inflict burn for every hit scored on an enemy. The C++ code would be responsible for registering a hit and sending a signal to blueprints called ‘OnHit’. In blueprints we could then roll a dice to see if we should inflict burn and then if that was successful, play a fire effect and add the burn status effect to the enemy.
